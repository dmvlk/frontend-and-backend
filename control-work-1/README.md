Магазин кофe

Swagger документация API: http://localhost:3000/api-docs



1. CSS-препроцессоры (практика №1)
SASS использован для стилизации карточки товара. С помощью переменных заданы основные цвета: бежевый для фона страницы, коричневый для заголовков и цены, песочный для акцентов и границ. Миксин button-soft применён для создания единообразных кнопок с закруглёнными углами и эффектом наведения.Готовая карточка товара отображается в файле practice-1/index.html.

2. Серверная часть на Node.js и Express (практики №2, №4, №5)
Express использован для создания REST API. В файле app.js настроены маршруты для работы с товарами:

GET /api/products — получение списка всех товаров (12 сортов кофе)

GET /api/products/:id — получение конкретного товара по ID

POST /api/products — добавление нового товара (с полями: название, категория, цена, описание, количество на складе)

PATCH /api/products/:id — обновление существующего товара

DELETE /api/products/:id — удаление товара

Middleware применены для:

парсинга JSON-запросов (express.json())

логирования всех запросов в консоль (метод, путь, статус, тело запроса)

обработки CORS для связи с фронтендом на порту 3001

Nanoid используется для генерации уникальных идентификаторов товаров (6 символов).



3. Фронтенд на React (практики №4, №5)
React использован для создания пользовательского интерфейса. Компоненты разбиты по папкам:

ProductItem.jsx — карточка отдельного товара

ProductsList.jsx — список всех карточек с проверкой на пустой массив

ProductModal.jsx — модальное окно для создания и редактирования товара

ProductsPage.jsx — главная страница, где происходит загрузка данных с API, управление состоянием и рендер компонентов

Axios используется для HTTP-запросов к бэкенду. В файле src/api/index.js создан экземпляр axios с базовым URL http://localhost:3000/api, настроены заголовки и экспортированы методы для каждого CRUD-действия.

SASS на фронтенде применён для стилизации страницы с товарами. Цветовая гамма полностью повторяет первую практику:

фон страницы: #f5ebe0 (бежевый)

карточки: #fff9f2 (тёплый белый)

заголовки и цена: #8b5a2b (коричневый)

акценты и границы: #d9b382 (песочный)

текст: #4a3b2c (мягкий коричневый)

Для карточек заданы скруглённые углы (24px), тень при наведении, фиксированная высота (580px) и 4 строки для описания с обрезанием и многоточием.

Логика работы с данными:

При загрузке страницы вызывается api.getProducts(), данные сохраняются в useState

При добавлении/редактировании открывается модальное окно, после отправки формы вызывается api.createProduct() или api.updateProduct(), состояние обновляется

При удалении вызывается api.deleteProduct() после подтверждения в window.confirm

Картинки для каждого товара сохранены в папке public/images с названиями на английском (например, brazil-sul-de-minas.webp). В компоненте ProductItem.jsx реализована функция маппинга русских названий товаров на имена файлов, что гарантирует подгрузку правильного изображения.

4. Тестирование API (практика №3)


Postman использован для тестирования всех эндпоинтов. Создана коллекция запросов:

GET /api/products — проверка получения всех 12 товаров (статус 200)

GET /api/products/1 — проверка получения конкретного товара по ID (статус 200)

GET /api/products/999 — проверка обработки несуществующего ID (статус 404)

POST /api/products — создание нового товара с телом JSON (статус 201)

DELETE /api/products/1 — удаление товара (статус 204)

Каждый запрос успешно протестирован.
Скриншоты каждого запроса с методом расположены practice-3/screenshots. Отчёт в формате Markdown с встроенными изображениями находится в practice-3/practice-3.md.

Swagger подключён через библиотеки swagger-jsdoc и swagger-ui-express. Готовая документация доступна по адресу http://localhost:3000/api-docs в интерактивном формате.
