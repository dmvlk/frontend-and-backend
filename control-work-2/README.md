Ссылка на репозиторий: https://github.com/dmvlk/frontend-and-backend


Swagger документация API: http://localhost:3000/api-docs



1. Аутентификация (практика №7)
Реализована регистрация и вход пользователей. Пароли хешируются bcrypt. Пользователи хранятся в оперативной памяти сервера.

Маршруты:
POST /api/auth/register - регистрация (email, first_name, last_name, password)
POST /api/auth/login - вход по email и паролю


2. JWT токены (практика №8)
Добавлена генерация JWT access-токена при входе. Токен возвращается в ответе и используется для авторизации. Добавлен защищённый маршрут GET /api/auth/me, возвращающий данные текущего пользователя по токену.

3. Refresh токен (практика №9)
При входе выдаются access токен и refresh токен. Refresh-токены хранятся в памяти сервера. Добавлен маршрут POST /api/auth/refresh для обновления пары токенов по refreshToken.

4. Хранения токенов (практика №10)
Токены хранятся в httpOnly cookies, что делает их недоступными для JavaScript и защищает от XSS-атак. На фронтенде настроен axios с withCredentials: true для автоматической отправки cookies. Добавлен interceptor для обработки ошибки 401 — при истечении accessToken автоматически отправляется запрос на /api/auth/refresh, после чего повторяется исходный запрос. 

5. RBAC (практика №11)
Роли:
user - просмотр и покупка товаров
seller - управление товарами
admin - управление пользователями

Маршруты:
GET /api/users - список пользователей (admin)
PUT /api/users/:id - редактирование пользователя (admin)
DELETE /api/users/:id - блокировка пользователя (admin)
POST /api/products - создание товара (seller    )
PUT /api/products/:id - редактирование товара (seller)
DELETE /api/products/:id - удаление товара (admin)

6. Хранение данных (практика №11)
Пользователи и товары хранятся в JSON файлах (data/users.json и data/products.json). При перезапуске сервера данные не теряются. При любых изменениях файлы обновляются.

7. Фронтенд
React приложение. Компоненты:
Login - вход и регистрация
ProductsPage - товары (разный интерфейс для разных ролей)
UsersPage - управление пользователями (только admin)
Cart - модалка корзины
