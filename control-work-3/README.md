### Service Worker и офлайн-доступ (практика №13)

Service Worker регистрируется при загрузке страницы и кэширует все статические ресурсы (HTML, CSS, JS, иконки, картинки). При повторных посещениях страница загружается из кэша, даже при отсутствии интернета. Список кэшируемых файлов указан в sw.js.

### Web App Manifest (практика №14)

Создан файл manifest.json, который содержит название приложения, цветовую тему, ориентацию экрана и набор иконок разных размеров (16x16, 32x32, 96x96, 128x128, 256x256, 512x512). Благодаря манифесту приложение можно установить на устройство как PWA — оно запускается в отдельном окне без адресной строки браузера.

### App Shell (практика №15)

Реализована архитектура App Shell: каркас приложения (шапка, навигация, подвал) кэшируется и загружается мгновенно, а контент подгружается динамически через fetch. Страницы Главная и О приложении лежат в отдельных HTML-файлах в папке content. Статика загружается по стратегии Cache First, динамический контент по стратегии Network First с фолбеком на кэш.

### HTTPS (практика №15)

С помощью утилиты mkcert сгенерированы самоподписанные сертификаты для локального HTTPS. Сертификаты (server.cert.pem, server.key.pem) используются при запуске клиентского сервера. Это необходимо для работы Service Worker и push-уведомлений.

### WebSocket (практика №16)

На сервере настроен Socket.IO для двусторонней связи в реальном времени. При добавлении нового товара сервер через WebSocket отправляет событие всем подключённым клиентам, и на их страницах появляется всплывающее сообщение без перезагрузки.

### Push-уведомления (практика №16)

Реализована отправка push-уведомлений через web-push с использованием VAPID-ключей. Пользователь подписывается на уведомления через кнопку Включить уведомления. Уведомления приходят в двух случаях:
- при добавлении нового товара (для всех подписанных пользователей)
- при оформлении заказа (только для пользователя, сделавшего заказ)

Уведомления работают даже при закрытом браузере благодаря обработчику push в Service Worker.

### Напоминания и распродажи (практика №17)

Продавец может создавать распродажи: выбрать категорию товаров, процент скидки, дату и время начала и окончания. В указанное время всем подписанным пользователям приходит push-уведомление о старте распродажи с кнопкой Напомнить через 5 минут. При нажатии на кнопку уведомление откладывается и придёт снова через 5 минут.

Цены на товары во время распродажи автоматически пересчитываются, старая цена зачёркивается, новая округляется до десятков. После окончания срока скидки цены возвращаются к исходным.

### Фронтенд (практики №13-17)

Приложение реализовано на React с компонентным подходом. Функционал:
- вход и регистрация
- просмотр каталога кофе (12 сортов)
- корзина и оформление заказа для роли user
- управление товарами (добавление, редактирование, удаление) для ролей seller и admin
- управление пользователями (просмотр, редактирование, блокировка) для роли admin
- создание распродаж для роли seller
- офлайн-доступ через Service Worker
- push-уведомления с возможностью отложить на 5 минут
